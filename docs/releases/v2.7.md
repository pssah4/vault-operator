# Obsilo v2.7 -- Cross-surface MCP, vault summary maintenance, plugin store compliance

The v2.7 line opens Obsilo up to other AI surfaces (Claude Desktop, ChatGPT, Perplexity), brings the Karpathy-style wiki-maintenance pattern into the vault, and lands the final round of plugin-store review-bot compliance fixes that block the Obsidian Community Plugin submission.

## Highlights

### Cross-surface AI workflow (EPIC-23)

External AI surfaces can now read and write Obsilo's memory and history layers via MCP. Four new remote tools land:

- `save_to_memory` -- fact persistence with source tagging
- `save_conversation` -- conversation as living document
- `recall_memory` -- cross-source memory retrieval
- `search_history` -- cross-source history search

Each conversation carries an origin surface tag (claude / chatgpt / perplexity / obsilo / other / unknown). The history sidebar gets source tabs for filtering. Sync mode is configurable per provider for the privacy trade-off. The legacy `update_memory` tool is deprecated, with a migration helper in the Settings tab.

### Living documents + cross-interface threads

Multiple `save_conversation` calls within 30 minutes from the same source append to a single thread `thread-YYYY-MM-DD-{6-hex}` instead of creating new conversations. Living documents are append-only.

### Vault-to-memory bridge

Vault notes can now be marked as a memory source. The `FrontmatterIndexer` watches changes and triggers the `SingleCallProcessor` so the agent's facts stay in sync with the vault. Hooks: `addNoteAsMemorySource`, `removeNoteAsMemorySource` (FEAT-03-25, ADR-109).

### Karpathy-style wiki maintenance (BA-25)

A five-phase pipeline for keeping vault summaries fresh, ingesting external sources into clusters, and detecting tension between linked notes:

- **Phase 1** -- KnowledgeDB schema v9 -> v10 (four new tables), auto-summary pipeline.
- **Phase 2** -- Tension detection (hybrid: cosine + LLM judge).
- **Phase 3** -- Pre-triage tool, auto-trigger detection on configurable frontmatter property.
- **Phase 4** -- Frontmatter-conflict detection.
- **Phase 5** -- Stage-3 job runner with token-budget enforcement and a top-hub block whose KV-cache lifecycle is bounded.

### Memory v2 stabilisation pass (Track 3)

Three impacts:

- IMP-03-17-01 -- `recall_memory` cosine NaN guard.
- IMP-03-18-01 -- `AgingService` daily scheduler.
- IMP-03-18-02 -- `DriftBus` throttle bypass.

### Architecture decisions

21 new ADRs (ADR-90 through ADR-110), all accepted and linked from arc42 section 9. Topics: cost-aware heuristics, KnowledgeDB v10, source identity, cluster half-life, frontmatter conflicts, MOC marker block, KV cache, pre-triage, tension detection, output mode, web search, stage-3 runner, MCP memory versioning, source-interface tagging, vault-to-memory bridge (supersedes ADR-87), living documents.

## v2.7.1 follow-up (2026-05-05)

### Plugin store review-bot compliance

The v2.7.0 release passed local lint with the project's brand allowlist, but the Obsidian Community Plugin review bot uses default plugin options without that allowlist. v2.7.1 brings every reported finding from PR #11394 to zero.

- **Stringification (no-base-to-string)** -- type guards instead of `String(unknown)` in three call sites: `AutoTriggerObserver.matchesValue` (frontmatter property comparison), `SemanticIndexService` tag lookup, `validateNewTaskInput` (four fields).
- **Unbound method** -- `DeepIngestPipeline` now wraps `TensionDetector.markerWorthy` in an arrow function so `this` stays bound.
- **Floating promise** -- `updateMemory` legacy telemetry call is prefixed with `void`.
- **Redundant union** -- `string | unknown` reduced to `unknown` in `executeVaultOp` (string is already a subtype).
- **Static-style disable** -- the cluster-stale Notice in main.ts no longer disables `obsidianmd/no-static-styles-assignment`. A new utility class `.agent-u-cursor-pointer` replaces the inline `el.style.setProperty('cursor', 'pointer')`.
- **Sentence case (29 strings)** -- locale strings and UI text rewritten to satisfy the bot's default brand/acronym list. "Obsilo" was dropped from settings/error/onboarding copy ("the agent" instead). The ChatGPT account block was reworded to avoid `ChatGPT`/`OS`/`Plus`/`Pro` tokens. The eleven BA-25 commands and notices in main.ts were translated from German to English. "MOC" was replaced with "map-of-content"/"hub" in the vault settings.

### Relay quota fix (FIX-14-03)

Two bugs surfaced when the Cross-surface MCP relay started seeing real-world traffic.

- **FIX-14-03-01** -- relay poll interval raised from 2s to 10s. Cloudflare Workers Free Plan caps requests at 100k/day per account; 2s polling produced 43.200 requests per day per open Obsidian instance, independent of actual MCP usage. With BRAT hot reloads and multi-device setups the cap was hit even on idle days. Poll interval and reconnect delays are now named constants for explicit cost tracking.
- **FIX-14-03-02** -- the poll loop's bare `catch {}` hid HTTP status, body, and stack. A `describeRequestError` helper builds a one-line diagnostic and a `redactToken` helper strips the relay token before logging. After three consecutive failures a single Notice surfaces the issue without devtools. AUDIT-005 H-2/H-3 still hold: every logged string runs through token redaction.
